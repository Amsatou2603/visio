✅ Cloudinary configuré

CAUSE: Les fixtures contiennent des chemins locaux (media/products/...)
       mais pas les entrées ProductImage dans la base de données.

✅ SOLUTION APPLIQUÉE
═══════════════════════════════════════════════════════════════════════

J'ai modifié load_initial_data.py pour créer automatiquement les 
ProductImage pour chaque produit en pointant vers Cloudinary.

Maintenant le script:
1. Crée le superuser
2. Crée le vendeur
3. Charge les produits
4. Crée les images des produits (NOUVEAU!)

🆘 SI LES IMAGES NE S'AFFICHENT TOUJOURS PAS
═══════════════════════════════════════════════════════════════════════

Problème 1: Images créées mais URL incorrecte
──────────────────────────────────────────────
→ Vérifiez que les 3 variables Cloudinary sont sur Render:
  CLOUDINARY_CLOUD_NAME=drt6c8efi
  CLOUDINARY_API_KEY=853275685212226
  CLOUDINARY_API_SECRET=ThoYi2ke_9uYDF-DbCYaDL9uKmY

Problème 2: Images non uploadées sur Cloudinary
────────────────────────────────────────────────
→ Exécutez localement: python upload_media.py
→ Vérifiez sur: https://console.cloudinary.com

Problème 3: "Des images existent déjà" mais elles sont vides
─────────────────────────────────────────────────────────────
→ Supprimez les images existantes dans l'admin
→ Redéployez pour les recréer

Problème 4: Erreur 404 sur les images
──────────────────────────────────────
→ Les noms de fichiers doivent correspondre exactement
→ Vérifiez les noms sur Cloudinary vs le mapping ci-dessus
