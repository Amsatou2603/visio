📊 PROBLÈME
═══════════════════════════════════════════════════════════════════════

Le build réussit mais vous ne voyez PAS "Running migrations" dans les logs.

CAUSE: Render n'utilise pas render.yaml ou build.sh
       Il utilise la configuration du Dashboard

═══════════════════════════════════════════════════════════════════════

✅ SOLUTION : CONFIGURER MANUELLEMENT LE DASHBOARD
═══════════════════════════════════════════════════════════════════════

ÉTAPE 1: Aller sur Render Dashboard
────────────────────────────────────────────────────────────────────────

1. https://dashboard.render.com
2. Cliquez sur: visio-backend-sp1h
3. Cliquez sur: Settings (menu de gauche)

ÉTAPE 2: Modifier le Build Command
────────────────────────────────────────────────────────────────────────

Cherchez la section "Build & Deploy"

Dans "Build Command", REMPLACEZ par:

pip install -r requirements.txt && python manage.py migrate --no-input --verbosity 2 && python manage.py collectstatic --no-input

ÉTAPE 3: Modifier le Start Command
────────────────────────────────────────────────────────────────────────

Dans "Start Command", REMPLACEZ par:

python manage.py migrate --no-input && gunicorn core.wsgi:application --bind 0.0.0.0:$PORT --log-level info

ÉTAPE 4: Vérifier Root Directory
────────────────────────────────────────────────────────────────────────

Dans "Root Directory", mettez:

visio_backend

ÉTAPE 5: Sauvegarder
────────────────────────────────────────────────────────────────────────

1. Cliquez sur "Save Changes" en bas
2. Render va redéployer automatiquement

ÉTAPE 6: Surveiller les Logs
────────────────────────────────────────────────────────────────────────

1. Cliquez sur "Logs" (menu de gauche)
2. Attendez de voir:

   ==> Building...
   Successfully installed Django-4.2.30 ...
   
   Running migrations:
     Applying contenttypes.0001_initial... OK
     Applying auth.0001_initial... OK
     Applying users.0001_initial... OK
     Applying products.0001_initial... OK
     Applying orders.0001_initial... OK
     Applying payments.0001_initial... OK
     Applying reviews.0001_initial... OK
     ...
   
   ==> Build successful!
   
   ==> Starting service...
   Running migrations:
     No migrations to apply.
   Booting worker with pid: 123
   Listening at: http://0.0.0.0:10000

ÉTAPE 7: Tester
────────────────────────────────────────────────────────────────────────

Attendez 2-3 minutes, puis:

python check_deployment.py

Ou testez:
✅ https://visio-backend-sp1h.onrender.com/health/
✅ https://visio-backend-sp1h.onrender.com/api/products/
✅ https://visio-backend-sp1h.onrender.com/admin/

🆘 SI ÇA NE MARCHE TOUJOURS PAS
═══════════════════════════════════════════════════════════════════════

Option 1: Clear Build Cache
────────────────────────────
Dashboard → Settings → "Clear build cache & deploy"

Option 2: Vérifier Neon Database
─────────────────────────────────
1. Allez sur: https://console.neon.tech
2. Vérifiez que la base est Active
3. Copiez le Connection String
4. Vérifiez qu'il correspond à DATABASE_URL sur Render

Option 3: Tester Quand Même
────────────────────────────
Parfois les migrations s'exécutent sans affichage.
Testez: https://visio-backend-sp1h.onrender.com/api/products/
Si pas d'erreur 500 → Les migrations ont marché!

💡 POURQUOI CETTE MÉTHODE FONCTIONNE
═══════════════════════════════════════════════════════════════════════

Render peut ignorer render.yaml si:
  - Le service a été créé manuellement
  - Des modifications ont été faites dans le Dashboard
  - Le fichier n'est pas à la racine du repo

La configuration manuelle via le Dashboard est TOUJOURS respectée!
